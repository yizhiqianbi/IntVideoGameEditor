# Agent 系统 Prompts 优化文档

## 📋 改进概述

### 优化前的问题
1. **Prompt 过于简略**：只要求输出 JSON，缺少具体指导
2. **缺少示例**：没有 few-shot 示例，AI 只能靠猜测
3. **角色设计不足**：bio 和 appearancePrompt 要求不明确
4. **镜头语言缺失**：videoPrompt 质量无法保证
5. **节奏感缺失**：剧本没有节奏和张力指导
6. **视觉描述模糊**：缺少具体化原则

### 优化后的改进

## 🎯 Draft Agent 改进

### 1. 角色设计原则
**新增内容：**
- ✅ bio 必须包含：年龄、职业、外貌特征、服装风格、性格、背景故事
- ✅ appearancePrompt 必须具体：发色发型、五官特征、服装细节、配饰、体态
- ✅ basePrompt 要突出视觉识别度

**示例对比：**
```
❌ 优化前：
"bio": "主角",
"appearancePrompt": "年轻人"

✅ 优化后：
"bio": "25岁，程序员，黑色短发戴眼镜，穿连帽衫，性格内向但执着",
"appearancePrompt": "年轻亚洲男性，黑色短发，黑框眼镜，灰色连帽衫，专注的眼神"
```

### 2. 镜头设计原则
**新增内容：**
- ✅ 明确景别运用（远景/全景/中景/近景/特写）
- ✅ 明确角度运用（平视/俯视/仰视/荷兰角）
- ✅ 明确运镜方式（固定/推拉/摇镜/跟拍/手持/无人机）
- ✅ videoPrompt 必须包含：景别、角度、运镜方式、光线、关键视觉元素

**示例对比：**
```
❌ 优化前：
"videoPrompt": "主角在写代码"

✅ 优化后：
"videoPrompt": "深夜办公室，中景，年轻程序员对着电脑，屏幕蓝光映在脸上，
桌上咖啡杯，杂乱的文件，窗外城市灯光"
```

### 3. 场景拆解原则
**新增内容：**
- ✅ 每个场景 3-8 秒的具体限制
- ✅ 一个场景只讲一件事
- ✅ 场景摘要四要素：谁 + 在哪 + 做什么 + 结果如何

### 4. 分支设计原则
**新增内容：**
- ✅ 分支选择要有明确意义（价值观/策略/情感导向）
- ✅ 预测结果要具体
- ✅ 提供了完整的分支示例

### 5. 结构完整性
**新增内容：**
- ✅ 质量检查清单
- ✅ 单根树结构验证
- ✅ 线性剧情处理

## 📝 Script Agent 改进

### 1. 剧本格式规范
**新增内容：**
- ✅ 明确的剧本结构模板
- ✅ 场景标题、镜头参数、视觉元素的标准格式
- ✅ 镜头切换标记

### 2. 镜头语言系统
**新增完整指南：**

#### 景别运用
- **远景 (ELS)**：建立环境，人物占画面 1/4 以下
- **全景 (LS)**：展现人物全身和环境关系，占画面 1/2
- **中景 (MS)**：展现人物半身（腰部以上），占画面 3/4
- **近景 (CU)**：展现人物面部和肩部，占画面 90%
- **特写 (ECU)**：局部细节（眼睛、手、物品）

#### 角度运用
- **平视**：客观、中性
- **俯视**：角色显得弱小、无助
- **仰视**：角色显得强大、威严
- **荷兰角**：倾斜画面，表现不安、混乱

#### 运镜方式
- **固定**：稳定、观察
- **推拉**：强调或放松
- **摇镜**：跟随动作或展示环境
- **跟拍**：增强动作感和代入感
- **手持**：真实感、紧张感
- **无人机**：宏大场面、上帝视角

### 3. 节奏控制系统
**新增内容：**
- ✅ 快节奏 vs 慢节奏的定义和使用场景
- ✅ 节奏变化的时机
- ✅ 张力营造技巧（信息差、时间压力、情绪反差、视觉隐喻）

### 4. 视觉描述技巧
**新增具体化原则：**
```
❌ 抽象描述：
"房间很乱"
"他很生气"

✅ 具体描述：
"衣服堆在椅子上，书本散落在地，外卖盒放在桌角"
"他猛地站起，椅子撞到墙上，脸涨得通红，额头青筋凸起"
```

**新增感官细节：**
- ✅ 视觉：颜色、光线、形状、动态
- ✅ 听觉：在画面描述中暗示声音来源
- ✅ 触觉：通过材质、温度来表现

**新增情绪视觉化：**
- ✅ 焦虑：频繁看表、来回踱步、咬指甲
- ✅ 悲伤：低头、背影、雨景、冷色调
- ✅ 愤怒：紧握拳头、咬牙、快速动作
- ✅ 喜悦：跳跃、笑容、明亮光线、暖色调

### 5. Few-shot 示例
**新增完整示例：**
- ✅ Draft Agent 示例："程序员的选择"（完整 6 场景，2 分支）
- ✅ Script Agent 示例：完整剧本，包含所有镜头语言

## 🔧 技术改进

### 代码组织
1. **创建独立文件**：`src/lib/agent-prompts.ts`
2. **模块化设计**：Draft 和 Script prompts 分离
3. **辅助函数**：`buildDraftUserPrompt` 和 `buildScriptUserPrompt`
4. **类型安全**：所有 prompts 都是常量，便于维护

### API 更新
1. **Draft API** (`/api/agent/draft`)：
   - 使用 `DRAFT_SYSTEM_PROMPT`
   - 使用 `buildDraftUserPrompt`

2. **Script API** (`/api/agent/script`)：
   - 使用 `SCRIPT_SYSTEM_PROMPT`
   - 使用 `buildScriptUserPrompt`

1. **添加更多示例**：不同类型的题材（爱情、悬疑、动作）

## 📝 使用说明

### 环境变量设置
```env
AGENT_API_KEY=your_volcengine_api_key
AGENT_API_MODEL=doubao-seed-2-0-lite-260215
```

### API 调用示例

#### Draft API
```typescript
const response = await fetch('/api/agent/draft', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    storyText: '你的故事内容',
    feedback: '可选的修订意见'
  })
});
```

#### Script API
```typescript
const response = await fetch('/api/agent/script', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    storyText: '你的故事内容',
    feedback: '可选的修订意见',
    draft: {...} // 可选的草案对象
  })
});
```

使用优化后的 Agent 系统，检查输出质量：

- [ ] 所有 ID 都是简短、稳定的 ASCII 字符串
- [ ] 每个场景时长 3-8 秒
- [ ] 每个 videoPrompt 都有：景别、角度、运镜、光线、关键视觉元素

- [ ] 场景形成完整的树形结构

### Script 检查清单
- [ ] 每个场景都有明确的景别、角度、运镜
- [ ] 视觉描述具体可感，不是抽象描述
